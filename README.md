# Auto Service

Информационная система для автоматизации автосервиса на Python и Flask.

## Возможности

- учет клиентов;
- учет автомобилей клиентов;
- справочник работ с нормо-часами и ставками;
- склад запчастей;
- создание заказ-нарядов;
- добавление работ и резервирование запчастей в заказ;
- списание фактически использованных запчастей;
- управление статусами заказов;
- печатная форма заказ-наряда в HTML и простой PDF.

## Запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
python main.py
```

API будет доступен по адресу `http://localhost:5000`.

## Публикация в облаке

Проект подготовлен для запуска на облачных платформах, которые поддерживают Python
и команду запуска из `Procfile`:

```bash
web: gunicorn main:app
```

Минимальные настройки сервиса:

- Build command: `pip install -r requirements.txt`
- Start command: `gunicorn main:app`
- Python: 3.12

Переменные окружения для облака:

- `SECRET_KEY` - любой длинный случайный ключ для сессий.
- `ADMIN_USERNAME` - логин администратора, например `admin`.
- `ADMIN_PASSWORD` - пароль администратора.
- `PUBLIC_APP_URL` - публичная ссылка на сайт, например `https://auto-service.example.com`.
- `DATABASE_URL` - необязательно. Если не указать, будет использоваться SQLite-файл `app/auto_service.db`.

После публикации приложение будет доступно по публичной ссылке. QR-код откройте по
адресу:

```text
https://ваш-домен/qr
```

Этот QR ведет на значение `PUBLIC_APP_URL`. Если переменная не задана, QR будет
создан на текущий адрес сайта, с которого открыта страница `/qr`.

## Доступ с телефона по QR-коду

1. Один раз запустите `install_once.bat`, чтобы установить зависимости.
2. Подключите компьютер и телефон к одной Wi-Fi сети.
3. Запустите `start_qr.bat`.
4. В браузере автоматически откроется QR-код. Отсканируйте его камерой телефона.
5. Не закрывайте окно запуска на компьютере, пока пользуетесь системой.

При первом запуске Windows может запросить разрешение для доступа Python к сети.
Разрешите доступ для частных сетей. QR-доступ внутри локальной сети не публикует
приложение в интернете.

## Основные маршруты

- `GET /api/clients` - список клиентов
- `POST /api/clients` - создать клиента
- `GET /api/cars` - список автомобилей
- `POST /api/cars` - создать автомобиль
- `GET /api/works` - список работ
- `POST /api/works` - создать работу
- `GET /api/parts` - список запчастей
- `POST /api/parts` - создать запчасть
- `GET /api/orders` - список заказ-нарядов
- `POST /api/orders` - создать заказ-наряд
- `GET /api/orders/<id>` - детальная карточка заказа
- `POST /api/orders/<id>/add_work` - добавить работу
- `POST /api/orders/<id>/add_part` - зарезервировать запчасть
- `POST /api/orders/<id>/use_part` - списать запчасть
- `PUT /api/orders/<id>/status` - изменить статус
- `GET /api/orders/<id>/document?format=html` - HTML-документ
- `GET /api/orders/<id>/document?format=pdf` - PDF-документ

## Примеры JSON

Создание клиента:

```json
{
  "name": "Иван Петров",
  "phone": "+7 999 111-22-33",
  "email": "ivan@example.com"
}
```

Создание автомобиля:

```json
{
  "license_plate": "A123BC125",
  "model": "Toyota Corolla",
  "vin": "JTDBR32E720000001",
  "year": 2018,
  "client_id": 1
}
```

Изменение статуса:

```json
{
  "status": "в работе"
}
```
